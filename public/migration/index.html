    Migration Guide: Obsidian/Quartz to Hugo#

    Quick Migration Steps#

    1. Backup Your Current Setup âœ…#

    # Backup your current content directory
cp -r content content-backup-$(date +%Y%m%d)

# Backup your hugo.toml
cp hugo.toml hugo.toml.backup

    2. Apply New Structure#

    # Remove old content
rm -rf content

# Move restructured content in place
mv restructured-content content

# Update configuration
cp content/hugo.toml hugo.toml

    3. Test Locally#

    # Start Hugo development server
hugo server -D

# Visit http://localhost:1313 to preview

    4. Deploy#

    # Build site
hugo --minify

# Deploy to your hosting (Cloudflare Pages, GitHub Pages, etc.)
git add .
git commit -m "Restructure site for better navigation"
git push

    Obsidian/Quartz Features Removed#

    Timeline Tags#

    <div class='ob-timelines' ...>

    These were Obsidian/Quartz specific and don’t render in Hugo. Consider replacing with date front matter:

    ---
date: 2024-01-07
---

    Wiki-Style Links#

    Before: [[Character Name]]
    After: [Character Name]()

    Hugo uses different internal linking syntax.

    Things to Update Manually#

    1. Session Notes Content#

    • Convert Obsidian-style links to Hugo relref
    • Remove timeline divs (already done for sample files)
    • Translate Indonesian text to English if desired

    2. Player Character Pages#

    • Some have complex HTML tables (class progression)
    • These work in Hugo but may need styling adjustments
    • Consider simplifying for web viewing

    3. NPC Pages#

    • The NPC directory was empty in your old structure
    • Populate content/campaigns/boundless-and-timeless/npcs/ as needed

    4. Images#

    • Obsidian image syntax: ![[image.png]]
    • Hugo image syntax: ![Alt text](image.png)
    • Place images in static/images/ directory

    Maintaining the Site#

    Adding New Sessions#

    1. Create file in appropriate party folder
    2. Follow naming convention: session-XX-title.md
    3. Update party index file with new session link

    Updating Characters#

    1. Edit character file in player-characters/
    2. Keep front matter updated
    3. Use Hugo/Markdown syntax

    Adding House Rules#

    1. Create new .md file in house-rules/
    2. Add link in house rules index
    3. Use hint boxes for important notes

    Troubleshooting#

    “Page not found”#

    • Check file names (lowercase, hyphens only)
    • Verify file path in links
    • Ensure front matter is valid

    “Links broken”#

    • Use relref shortcode for internal links
    • Path should be relative to content/
    • Don’t include file extension in relref

    “Navigation looks wrong”#

    • Check weight parameter in front matter
    • Lower numbers appear first
    • Default weight is 0

    “Theme not loading”#

    • Verify Hugo Book theme is installed
    • Check theme = "book" in hugo.toml
    • Run hugo mod get -u to update

    Need Help?#

    Common questions:

    Q: Can I keep my old structure?
    A: Yes, but the new structure is more scalable and player-friendly.

    Q: Will old links break?
    A: Yes, but you can set up redirects or update links systematically.

    Q: Can I customize the appearance?
    A: Absolutely! Hugo Book theme is highly customizable. Add custom CSS in static/custom.css.

    Q: How do I translate session notes?
    A: Session notes are still in Indonesian. You can gradually translate them or keep them as-is for Indonesian players.

    Q: What about images from Obsidian?
    A: Place images in static/images/ and update links to /images/filename.png
