    ğŸ¯ Key Improvements#

    1. Clear Navigation Hierarchy#

    • Homepage introduces your GM portfolio
    • Main sections are logically organized
    • Each section has an index page explaining its contents

    2. Player-Friendly Organization#

    • Campaign information is easy to find
    • Session notes are organized by party
    • House rules are separated into digestible sections
    • Character files use URL-friendly names

    3. Consistent Naming Conventions#

    • All files use lowercase with hyphens (e.g., alizar-valts.md)
    • No spaces or special characters in filenames
    • Session files are numbered for easy ordering

    4. Hugo-Book Theme Optimization#

    • Proper use of weight parameter for page ordering
    • Collapsible sections for better navigation
    • Hint boxes (info, warning, tip) for emphasis
    • Internal linking using relref shortcode

    ğŸš€ Getting Started#

    Installation#

    1. Make sure Hugo is installed:

      hugo version

    2. Replace your old content directory with this new structure:

      # Backup your old content first!
mv content content-backup
mv restructured-content content

    3. Copy the new hugo.toml:

      cp content/hugo.toml hugo.toml

    4. Run Hugo locally to test:

      hugo server -D

    5. Visit http://localhost:1313 to see your site!

    Adding New Session Notes#

    1. Navigate to the appropriate party folder:

      content/campaigns/boundless-and-timeless/session-notes/cestlavie-party/

    2. Create a new file following the naming convention:

      session-14-title-of-session.md

    3. Use this template:

      ---
title: "Session 14: Title of Session"
date: 2026-01-07
---

## Characters Present
- [Character Name]()

## Session Overview
Brief summary here...

## Key Learnings
- Important discovery 1
- Important discovery 2

## NPCs Met
- **NPC Name** - Description

## Detailed Recap
Full session details...

    Updating Player Characters#

    1. Navigate to:

      content/campaigns/boundless-and-timeless/player-characters/

    2. Edit the character’s markdown file

    3. Keep the front matter updated:

      ---
name: Character Name
class: Class - Subclass
race: Race
level: X
player: Player Name
ac: "X"
hp: X
---

    Adding New House Rules#

    1. Navigate to:

      content/house-rules/

    2. Create a new markdown file with an appropriate name

    3. Add to the house rules index file to link it

    ğŸ¨ Customization#

    Changing Theme Colors#

    The Hugo Book theme supports custom CSS. Create a file:

    static/custom.css

    Then reference it in your hugo.toml:

    [params]
  BookCustomCSS = ["/custom.css"]

    Adding a Logo#

    1. Place your logo image in:

      static/logo.png

    2. Enable it in hugo.toml:

      [params]
  BookLogo = '/logo.png'

    Customizing the Menu#

    Edit the hugo.toml file under [menu] section to add or remove menu items.

    ğŸ“Š Weight System#

    Hugo Book uses weight to order pages:

    • Lower numbers appear first
    • 10, 20, 30, etc. leaves room for insertions
    • Set in front matter: 
      ---
weight: 20
---

    ğŸ”— Internal Linking#

    Use Hugo’s relref shortcode for internal links:

    [Link Text]()

    Examples:

    [House Rules](/house-rules/)
[Alizar's Character](/campaigns/)

    ğŸ“¦ Deployment#

    Cloudflare Pages#

    1. Connect your GitHub repository
    2. Build command: hugo --minify
    3. Build output directory: public
    4. Environment variables:
      • HUGO_VERSION: 0.121.0 (or your version)

    GitHub Pages#

    1. Use the Hugo GitHub Actions workflow
    2. Push to your repository
    3. Site deploys automatically

    ğŸ†˜ Troubleshooting#

    Pages Not Showing Up#

    • Check file naming (lowercase, hyphens, .md extension)
    • Verify front matter is valid YAML
    • Check weight parameter for ordering

    Links Broken#

    • Use relref shortcode for internal links
    • Don’t include .md extension in links
    • Paths are relative to content directory

    Theme Issues#

    • Ensure Hugo Book theme is installed
    • Check hugo.toml theme setting
    • Run hugo mod get -u to update theme
